# Frontend 가이드라인

## 디자인 철학
심플하고 미니멀. 학습에 집중할 수 있도록 불필요한 시각적 요소를 최소화한다.
화려한 애니메이션, 그라디언트, 복잡한 레이아웃 금지.

## 컬러 팔레트
Tailwind CSS 기본 색상만 사용.

| 용도 | 색상 |
|------|------|
| 배경 | `gray-50` |
| 카드/패널 | `white` |
| 기본 텍스트 | `gray-900` |
| 보조 텍스트 | `gray-500` |
| 테두리 | `gray-200` |
| 주요 액션 버튼 | `gray-900` (어두운 단색) |
| 정답 피드백 | `green-600` |
| 오답 피드백 | `red-500` |
| 북마크 — 중요 | `red-500` |
| 북마크 — 어려움 | `orange-400` |
| 북마크 — 나중에 | `blue-400` |

## 레이아웃
- 최대 너비: `max-w-3xl` (모바일이 아닌 PC 기준, 가독성을 위해 너무 넓지 않게)
- 중앙 정렬
- 좌우 패딩: `px-4`
- 상단 네비게이션 바: 심플한 텍스트 링크만 (아이콘 최소화)

## 네비게이션 구조
```
[주택관리사 기출]   문제 조회  퀴즈  오답노트  통계  북마크
```

## 타이포그래피
- 폰트: 시스템 기본 폰트 (별도 웹폰트 로드 금지)
- 문제 텍스트: `text-base` (읽기 편한 크기 유지)
- 보기 텍스트: `text-sm`
- 제목: `font-semibold` (bold 과용 금지)

## 컴포넌트 원칙
- 버튼: 외곽선 버튼(`border`) 또는 단색 버튼만 사용. 그림자 효과 금지.
- 카드: 흰 배경 + 얇은 테두리(`border border-gray-200`). 그림자 최소화(`shadow-sm`만 허용).
- 모달/팝업: 꼭 필요한 경우만. 가능하면 인라인 UI로 대체.
- 로딩 상태: 스피너 대신 텍스트("처리 중...") 또는 단순 펄스 애니메이션.

## 페이지별 레이아웃 패턴

### 목록 페이지 (문제 조회, 오답노트, 북마크)
```
[필터 영역]
[검색창]
[결과 수]
[리스트 아이템]
[리스트 아이템]
...
```

### 상세 페이지 (문제 상세, 오답 상세)
```
[뒤로가기]
[메타 정보: 연도 / 과목 / 단원]
[문제 텍스트]
[보기 목록]
[액션 버튼들]
```

### 퀴즈 진행 페이지
```
[진행 상황: N / 전체]
[문제 텍스트]
[보기 목록]
[제출 버튼]
```

## 금지 사항
- 다크모드 구현 금지 (복잡도 증가)
- 커스텀 CSS 파일 생성 금지 (Tailwind 클래스만 사용)
- 외부 UI 라이브러리 도입 금지 (shadcn/ui, MUI, Chakra 등)
- 차트는 `recharts`만 허용 (학습 통계 페이지 전용)
