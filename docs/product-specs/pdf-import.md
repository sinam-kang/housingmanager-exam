# PDF 임포트

## 사용자 플로우
1. 앱에서 연도 폴더를 선택
2. 앱이 파일명 패턴으로 파일 자동 분류
3. 파싱 → 정답 매칭 → LLM 단원 태깅 → DB 저장
4. 임포트 결과 요약 표시 (성공 N문제 / 실패 N문제)

## 폴더/파일 구조 규칙

```
연도/
├── 1차 1교시 문제.pdf    → 1차, 1교시, 객관식 (과목 헤더로 민법/회계 구분)
├── 1차 2교시 문제.pdf    → 1차, 2교시, 객관식 (시설개론)
├── 1차 정답.pdf          → 1차 전체 정답 (1교시+2교시 합본)
├── 2차 시험 문제.pdf     → 2차, 객관식+주관식 혼합 (과목 헤더로 관계법규/관리실무 구분)
├── 2차 객관식 정답.pdf
└── 2차 주관식 정답.pdf
```

## 파싱 규칙

### 문제 파일
- 과목 헤더(예: "제1과목 민법") 기준으로 과목 경계 구분
- 문제 번호(예: "1.", "2.") 기준으로 문제 단위 분리
- 객관식: 보기 ①②③④⑤ 추출
- 주관식: 보기 없음, 문제 텍스트만 추출

### 정답 파일
- 문제 번호 + 정답(번호 또는 단답 텍스트) 쌍으로 추출
- 1차 정답.pdf: 1교시/2교시 구분자 기준으로 분리 후 각각 매칭
- 주관식 정답: 단답형 (숫자 또는 단어)

## 추출 데이터 (문제 1개 단위)

| 필드 | 설명 |
|------|------|
| 연도 | 폴더명에서 추출 |
| 시험차수 | 1차 / 2차 |
| 교시 | 1교시 / 2교시 / null(2차) |
| 과목 | 민법 / 회계 / 시설개론 / 관계법규 / 공동주택관리실무 |
| 문제번호 | 정수 |
| 문제유형 | 객관식 / 주관식 |
| 문제텍스트 | 문제 본문 |
| 보기 | ①②③④⑤ 배열 (객관식만) |
| 정답 | 객관식: 1~5 / 주관식: 단답 텍스트 |
| 단원 | LLM 자동 태깅 |

## 단원 태깅 (LLM)
- 임포트 완료 후 배치로 LLM 호출
- 입력: 과목명 + 문제 텍스트
- 출력: 과목 내 단원명
- 태깅 실패 시 "미분류"로 저장 → 사용자가 나중에 수동 수정 가능

## 오류 처리
- 파싱 실패 문제: 건너뛰고 실패 목록에 표시
- 정답 미매칭 문제: 정답 없이 저장 후 목록에 표시
- 중복 임포트(같은 연도): 경고 표시 후 덮어쓰기 / 건너뛰기 선택
