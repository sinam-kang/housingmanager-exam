# Core Beliefs

이 프로젝트의 모든 설계 결정의 바탕이 되는 원칙.
새로운 기능을 추가하거나 설계를 바꿀 때 이 원칙에 맞는지 먼저 확인한다.

## 1. 도구는 학습을 방해하지 않는다
앱을 사용하는 법을 배우는 데 시간을 쓰면 안 된다.
처음 켠 사람도 5분 안에 문제를 풀기 시작할 수 있어야 한다.

## 2. 데이터는 사용자 것이다
문제 데이터와 학습 기록은 사용자 PC에만 저장된다.
클라우드 전송, 외부 서버 저장 금지.

## 3. 단순함이 완성도다
기능이 많은 것보다 있는 기능이 잘 동작하는 것이 낫다.
스펙에 없는 기능은 추가하지 않는다.

## 4. 실패는 조용히 처리한다
PDF 파싱 실패, LLM 오류 등 부분적인 실패가 전체를 멈춰서는 안 된다.
실패한 것만 알려주고, 나머지는 계속 동작한다.

## 5. 비개발자가 쓸 수 있어야 한다
터미널을 모르는 사람도 `setup.sh`, `start.sh` 두 개만 알면 된다.
설정 파일을 직접 편집하게 만들지 않는다.
