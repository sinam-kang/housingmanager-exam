# 구현 계획

에이전트는 아래 순서대로 구현한다. 각 단계가 완료된 후 다음 단계로 넘어간다.
앞 단계가 뒤 단계의 기반이 되므로 순서를 바꾸지 않는다.

---

## Phase 1 — 프로젝트 기반 설정
**목표:** 에이전트와 사용자 모두 실행 가능한 빈 앱 골격 완성

- [ ] 디렉토리 구조 생성 (`backend/`, `frontend/`, `data/`)
- [ ] 백엔드 초기화 (`uv init`, FastAPI, SQLModel, pdfplumber, anthropic 패키지 설치)
- [ ] 프론트엔드 초기화 (`Next.js`, TypeScript, Tailwind CSS)
- [ ] SQLite DB 연결 및 기본 테이블 생성
- [ ] `setup.sh` 작성 (최초 1회 설치 스크립트)
- [ ] `start.sh` 작성 (백엔드 + 프론트엔드 동시 실행)
- [ ] `backend/.env.example` 작성 (ANTHROPIC_API_KEY 안내 포함)
- [ ] 완료 기준: `./start.sh` 실행 후 브라우저에서 빈 화면 정상 접속

---

## Phase 2 — PDF 임포트
**목표:** PDF 파일에서 문제 데이터를 추출하여 DB에 저장
**스펙:** `docs/product-specs/pdf-import.md`

- [ ] DB 스키마 완성 (`questions` 테이블 및 연관 테이블)
- [ ] PDF 파서 구현 (`backend/services/pdf_parser.py`)
  - 문제 파일 파싱 (문제 텍스트, 보기 추출)
  - 정답 파일 파싱 (번호-정답 쌍 추출)
  - 문제-정답 번호 매칭
- [ ] LLM 단원 태깅 구현 (`backend/services/llm.py`)
- [ ] 임포트 API 엔드포인트 (`backend/routers/import.py`)
- [ ] 임포트 UI 구현
  - 연도 폴더 선택
  - 진행 상태 표시
  - 결과 요약 (성공/실패 문제 수)
- [ ] 완료 기준: 실제 PDF 파일 임포트 후 DB에 문제 데이터 저장 확인

---

## Phase 3 — 문제 조회/검색
**목표:** 임포트된 문제를 목록으로 보고 검색할 수 있다
**스펙:** `docs/product-specs/question-browse.md`

- [ ] 문제 목록 API 엔드포인트
- [ ] 키워드 검색 API
- [ ] 문제 목록 페이지 UI
- [ ] 문제 상세 페이지 UI (정답 토글 포함)
- [ ] 완료 기준: 목록에서 문제 검색 및 상세 확인 가능

---

## Phase 4 — 필터
**목표:** 연도/과목/단원/문제유형으로 문제를 좁혀볼 수 있다
**스펙:** `docs/product-specs/filters.md`

- [ ] 필터 API 파라미터 추가 (문제 목록 API에 통합)
- [ ] 공통 필터 컴포넌트 UI (`frontend/components/Filter.tsx`)
  - 연도, 시험차수, 과목, 단원(계층형), 문제유형
  - 북마크/오답 이력 토글
- [ ] 문제 목록 페이지에 필터 연동
- [ ] 완료 기준: 필터 조합으로 원하는 문제만 표시 확인

---

## Phase 5 — 퀴즈 모드
**목표:** 선택한 범위의 문제를 퀴즈 형식으로 풀 수 있다
**스펙:** `docs/product-specs/quiz-mode.md`

- [ ] 퀴즈 세션 DB 스키마 (`quiz_sessions`, `quiz_answers` 테이블)
- [ ] 퀴즈 API 엔드포인트 (세션 생성, 답안 제출, 결과 조회)
- [ ] 퀴즈 시작 설정 화면 UI (범위 선택, 출제 방식 선택)
- [ ] 퀴즈 진행 화면 UI (문제 표시, 답 선택, 즉시 피드백)
- [ ] 퀴즈 종료 화면 UI (결과 요약)
- [ ] 오답 발생 시 오답노트 자동 저장 연동
- [ ] 완료 기준: 퀴즈 시작부터 종료까지 전체 흐름 동작 확인

---

## Phase 6 — 오답노트 + 해설 보기
**목표:** 틀린 문제를 모아보고 LLM 해설을 요청할 수 있다
**스펙:** `docs/product-specs/wrong-answer-notes.md`

- [ ] 오답노트 API 엔드포인트 (목록, 상세, 제거)
- [ ] 해설 생성 API 엔드포인트 (LLM 호출 및 저장)
- [ ] 오답노트 목록 UI (정렬 선택 포함)
- [ ] 오답 상세 UI ("해설 보기" 버튼, 해설 표시)
- [ ] 완료 기준: 오답노트에서 해설 생성 및 저장 확인

---

## Phase 7 — 학습 통계/진도
**목표:** 학습 현황을 한눈에 파악할 수 있다
**스펙:** `docs/product-specs/study-stats.md`

- [ ] 통계 API 엔드포인트 (진도, 과목별/연도별 정답률, 취약 단원, 퀴즈 기록, 연속 학습일)
- [ ] 통계 대시보드 UI
- [ ] 완료 기준: 통계 화면에서 모든 항목 정상 표시 확인

---

## Phase 8 — 북마크
**목표:** 문제에 태그를 붙여 북마크하고 모아볼 수 있다
**스펙:** `docs/product-specs/bookmarks.md`

- [ ] 북마크 DB 스키마 및 API 엔드포인트
- [ ] 북마크 버튼 컴포넌트 (태그 선택 팝업 포함)
- [ ] 북마크 목록 페이지 UI
- [ ] 문제 목록/상세/퀴즈/오답노트에 북마크 버튼 연동
- [ ] 완료 기준: 전체 화면에서 북마크 등록/해제/조회 동작 확인
