# Quality Score

에이전트가 각 기능의 완성도를 판단하는 기준.

## 전체 기준

| 항목 | 기준 |
|------|------|
| 기능 완성도 | 스펙 파일의 모든 요구사항 충족 |
| UI 언어 | 모든 텍스트 한국어 |
| 에러 처리 | 모든 에러 상황에서 사용자 친화적 한국어 메시지 표시 |
| 로딩 상태 | 1초 이상 걸리는 작업은 반드시 진행 상태 표시 |
| 실행 방법 | `start.sh` 하나로 실행 가능 |

## 기능별 품질 기준

### PDF 임포트
- 파싱 성공률 95% 이상 (100문제 기준 5개 이하 실패 허용)
- 실패한 문제는 목록에 명확히 표시
- 중복 임포트 시 데이터 손실 없음
- 임포트 진행 상태 실시간 표시

### 문제 조회/검색
- 키워드 검색 결과 0.5초 이내 응답
- 필터 적용 즉시 반영

### 퀴즈 모드
- 답 제출 후 즉시 맞음/틀림 피드백
- 퀴즈 도중 새로고침해도 세션 유지
- 오답 발생 시 오답노트 자동 저장 누락 없음

### 오답노트 + 해설
- LLM 해설 생성 실패 시 재시도 가능
- 한 번 생성된 해설은 재호출 없이 즉시 표시

### 학습 통계
- 통계 페이지 로딩 1초 이내
- 퀴즈 완료 후 통계에 즉시 반영

### 북마크
- 어느 화면에서든 북마크 등록/해제 즉시 반영

## 완료 선언 조건
아래를 모두 만족해야 해당 Phase를 완료로 간주한다:
- [ ] 스펙의 모든 기능이 동작한다
- [ ] 에러 상황(네트워크 끊김, 잘못된 입력 등)에서 앱이 죽지 않는다
- [ ] `start.sh`로 실행 후 처음 사용하는 사람도 기능을 찾을 수 있다
