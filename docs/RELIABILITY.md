# Reliability

## 핵심 원칙
데이터 손실 없음. 임포트한 문제와 학습 기록은 어떤 상황에서도 보존되어야 한다.

## 데이터 안전성

### SQLite DB
- DB 파일은 `data/db.sqlite` 한 곳에만 저장
- 임포트 작업은 트랜잭션으로 처리 — 중간에 실패해도 부분 저장 없이 롤백
- 앱 종료 시 DB 연결 정상 종료

### PDF 파싱
- 파싱 실패는 전체 임포트를 중단시키지 않는다
- 실패한 항목만 수집하여 결과 화면에 표시
- 원본 PDF 파일은 절대 수정하지 않는다

## 에러 복구

### 백엔드 에러
- 500 에러 발생 시 프론트엔드에 한국어 안내 메시지 표시
- LLM 호출 실패 시 재시도 버튼 제공 (자동 재시도 없음 — 비용 발생 방지)

### 프론트엔드 에러
- 페이지 단위 에러 바운더리 적용 — 한 페이지 에러가 전체 앱을 죽이지 않도록
- API 응답 없음 시 "서버에 연결할 수 없습니다. start.sh가 실행 중인지 확인하세요." 표시

### 퀴즈 세션
- 퀴즈 도중 새로고침 또는 브라우저 재시작 시 마지막 문제부터 재개 가능
- 세션 데이터는 브라우저 localStorage가 아닌 DB에 저장

## 성능 기준
- 문제 목록 페이지 초기 로딩: 1초 이내
- 검색/필터 응답: 0.5초 이내
- 통계 페이지 로딩: 1초 이내
- LLM 해설 생성: 제한 없음 (사용자가 버튼을 눌러 시작하므로 대기 인지)
